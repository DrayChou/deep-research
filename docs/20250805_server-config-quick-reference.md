## 告警阈值

| 指标 | 警告 | 严重 | 处理建议 |
|------|------|------|----------|
| CPU | >70% | >85% | 扩容或优化 |
| 内存 | >80% | >90% | 增加内存 |
| 磁盘 | >80% | >95% | 清理或扩容 |
| 响应时间 | >3秒 | >10秒 | 检查API和DB |

## 扩容时机

- **CPU持续>70%**: 考虑垂直扩容
- **内存持续>80%**: 增加内存或优化
- **响应时间>5秒**: 检查瓶颈，准备扩容
- **用户增长>50%**: 提前规划扩容方案
